# .claude/bin: 開発補助スクリプトの方針

`.claude/bin/` 配下は、開発者向けの補助スクリプト（git worktree 操作など）を置く場所です。

本 README は横断方針のみを持ち、個別スクリプトの説明・索引は持ちません。スクリプトを書く・直すときの方針をまとめます。

## 短く保つ

簡潔に読めることを最優先し、「読んで分かる」長さに保ちます。各スクリプトの目的・挙動は先頭コメントに書きます。

多くは手入力していたコマンドのスクリプト化なので、いま必要でない機能を先回りして作り込みません。usage やヘルプの作り込み、過剰な入力検証は持たず、引数ミスは `set -u` 等で素直に落とします。

オプション・引数はまず最小限にし、汎用化・抽象化は必要になってから入れます。機能を増やす際には、別スクリプトに分けることも検討します。

## エラーは握り潰さない

`set -euo pipefail` で fail-fast にします。失敗の出所が下層コマンド（git 等）のメッセージから判別できれば十分で、整形や説明の作り込みはしません。

真因は消しません。`2>/dev/null` で抑制するのは、より正確なエラーを別に出せる場合に限ります（例: フォールバックの誤メッセージは捨て、最初の根本原因を表示する）。下層メッセージだけでは何のスクリプトの失敗か判別できない箇所に限り、`<scriptname>:` 接頭辞で文脈を足します。

## 環境前提・破壊的操作

実際の開発環境にだけ最適化します。旧バージョンやレア環境向けの分岐は持たず、前提を満たさなければ素直に失敗させます。

取り返しのつかない操作は条件を絞り、可能なら下層の安全機構に委ねます（例: worktree 削除は、未コミット作業があれば git に拒否させる）。
