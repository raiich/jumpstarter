# レビュー深刻度基準

レビュー結果の指摘を深刻度で分類する際の基準。コード・設計・ドキュメントのレビューで共通して使用する。

## 基準

- **Critical**: 即修正必須
  - コード: 実行時エラー、データ損失、セキュリティ脆弱性
  - 設計: 根本的な問題（実装前に解決必須）
  - ドキュメント: 利用者が誤った使い方をする原因（間違ったコード例、存在しないAPI、誤った手順）
- **Major**: 早期修正推奨
  - コード: 重要な設計問題、リソースリーク
  - 設計: 重要だが回避策がある（実装前に検討推奨）
  - ドキュメント: 重要な情報の欠落、曖昧な説明
- **Minor**: 改善が望ましいが機能には影響しない
  - 冗長な説明、構成の改善、実装後でも対応可能な改善
- **Info**: 参考情報・提案・好み・コードスタイル

## 出力フォーマット例

```markdown
| # | 深刻度 | 観点 | 指摘内容 | 該当箇所 |
|---|--------|------|----------|----------|
| 1 | Critical | API 設計 | 戻り値が null になりうるがドキュメント化されていない | foo.ts:42 |
```
