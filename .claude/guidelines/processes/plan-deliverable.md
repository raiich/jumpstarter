# プランファイル冒頭テンプレート: 成果物セクション

Plan mode のプランファイル冒頭に必ず記載する「成果物」セクション。

## 目的

Plan mode 終了後にコンテキストが失われた場合でも、成果物の種類を正しく判断できるようにする。

## テンプレート

```markdown
## 成果物
- 種類: [成果物の種類]（例: Design Doc、OKR、テストケース設計）
- 保存先: [絶対パスまたは相対パス]
- ⚠️ [注意事項・スコープ外の責務]
```

## 例

### Design Doc の場合

```markdown
## 成果物
- 種類: Design Doc（マークダウンドキュメント）
- 保存先: .local/docs/features/[名前]/design.md
- ⚠️ この段階ではコード実装を行わない
```

### OKR の場合

```markdown
## 成果物
- 種類: OKRドキュメント（マークダウン）
- 保存先: .local/docs/okr/{年度}.md
```
