# 既存成果物のレビュー

既に作成済みのコード・ドキュメント・設計等を対象に問題を抽出する場面に適用する。記述スタイルガイドライン・迎合しないルールにも従う。以下は既存成果物のレビュー固有の事項。

## 指摘出力時

自分で指摘を出す際は [review-severity.md](../references/review-severity.md) の Critical/Major/Minor/Info を用いる。

## 深いレビューが必要なとき

多角的な観点（ドキュメント品質・コード設計・テスト品質）からの体系的なレビューが必要な場合は、`hyper-think` スキルの review モードを用いる。
