# 用語ガイドライン

## 「技術」の使い方

このプロジェクトでは「技術」を問題を解決するための体系的な手法・技法の意味で使い、特定のソフトウェア・サービス・ツールとは区別する。

### 区別

| 分類 | 例 |
|------|-----|
| 技術 | パケットルーティング、名前解決、動的計画法、転置インデックス、コンテナ隔離 |
| ソフトウェア/ツール | Docker, Rails, AWS, Hadoop, Git |
| 仕様/プロトコル | TCP/IP, HTTP, SIP |

### 表現例

- ❌ 「Dockerという技術」「React技術を導入」
- ✅ 「コンテナ技術（実装: Docker）」「コンポーネントベースのUI構築手法（React が採用）」

### 原則

- ツール名を「技術」と呼ばない
- 背後にある手法・原理を「技術」として記述し、ツールはその実装として言及する
