# アウトカム vs アウトプット

成果物（アウトプット）ではなく成果（アウトカム）で目標・指標を記述する原則。

## 定義

- **アウトプット（成果物）**: 作ったもの、リリースしたもの
- **アウトカム（成果）**: その結果として変わったこと、生まれた価値

## 変換例

| アウトプット（❌） | アウトカム（✅） | 変換の考え方 |
|------------------|----------------|-------------|
| CI/CD パイプラインを構築する | デプロイ頻度を週 1 回 → 日次に向上 | パイプラインの目的はデプロイ高速化 |
| モニタリングダッシュボードを作成する | 障害検知から対応開始までの時間を X 分 → Y 分に短縮 | ダッシュボードの目的は障害の早期発見 |
| ドキュメントを整備する | 新メンバーのオンボーディング期間を X 週 → Y 週に短縮 | ドキュメントの目的は立ち上がり支援 |
| テストカバレッジを 80% にする | リリース後の不具合発生率を X% → Y% に低減 | カバレッジは手段、品質が目的 |

## 変換の問いかけ

「それを達成すると、自分や顧客にとって何が変わりますか？」

## 適用場面

- OKR の Key Result 記述
- 機能要件の「Why」セクション
- タスク粒度の議論（単位機能 vs 価値）
