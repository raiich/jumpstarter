# Fix Well

修正系の指示を受けたとき、曖昧なら解釈を 1 つに確定し、指摘箇所だけを場当たり的に直さず、影響範囲・同類問題・整合性を先に洗い出して 1 回でまとめて修正するスキル。

## 実行例

明確な修正は影響範囲を洗い出して一括で:

```
❯ /fix-well user_id を account_id にリネームして

● fix-well を実行します。影響範囲を先に洗い出して 1 回でまとめて修正します。

  ## 修正完了: user_id → account_id

  - 定義: src/models/user.ts:42
  - 参照: src/api/auth.ts, src/api/users.ts（5 箇所）
  - 呼び出し元: src/components/*.tsx（8 箇所）
  - ドキュメント: docs/api.md, README.md

  ## 調査したが該当しなかった範囲

  - src/legacy/*: 古い別概念
  - migrations/*: DB カラム名は別タスク
```

曖昧な修正は着手前に解釈を確定:

```
❯ /fix-well このあたり整理して

● 対象・スコープが曖昧なので先に解釈を確定します。

  <ここでユーザーへヒアリング>

  ## 着手前の理解

  - **やること**: src/utils 配下の同種ヘルパー関数を統合・命名統一
  - **スコープ**: 含む: utils/* / 含まない: lib/*, テストヘルパー
  - **方針**: re-export を 1 ファイル経由に統一して呼び出し元への影響を最小化
  - **前提**: ✅ utils を直接 import している箇所が 27 個
  - **推測**: ⚠️ helpers/ サブディレクトリも同種関数（命名から推測）
  - **仮定**: ❓ deprecated 関数の互換 alias を残すか
```
