# ギャップ分析

コードベースの特性と現在の `.claude/` 設定を突き合わせ、カバーされていない領域を特定して改善を提案します。

ユーザーのフィードバックに依存せず、AI が能動的に「足りないもの」を見つける逆提案スキルです。

## 実行手順

### 1. コードベースの特性スキャン

以下を簡易的に把握する（深掘りしすぎない）:

- 言語・フレームワーク
- ディレクトリ構造の特徴
- テスト構成（テストフレームワーク、テストファイルの配置）
- CI/CD の有無と構成
- 主要な外部依存
- ドキュメント構成（README、docs/、設計ドキュメントの有無）
- Makefile / タスクランナー / 自動化スクリプトの有無

### 2. 現在の .claude/ 設定の確認

- `rules/`（実体は `.github/instructions/`）の内容
- `skills/` の一覧と各スキルの目的
- `agents/` の一覧と各エージェントの役割
- `settings.local.json` の permissions

### 3. ギャップの特定

コードベースの特性に対して、以下の観点でカバーされていない領域を探す:

- **ルールの不足**: コードベースにパターンがあるのに、それを規定するルールがない
  - 例: テストコードがあるのにテスト規約がない
  - 例: 特定の命名規則が暗黙に使われているがルール化されていない
  - 例: エラーハンドリングのパターンが統一されていない
- **スキルの不足**: 繰り返し行われる作業がスキル化されていない
- **エージェントの不足**: 必要な専門的視点が欠けている
- **手順・自動化の不足**: 必要な手順の記載やスクリプトが存在しない
  - 例: セットアップ手順がない
  - 例: ビルド・テストの実行方法が書かれていない
  - 例: デプロイやリリースの手順が存在しない
- **設定の不整合**: 既存の設定同士の矛盾や、コードベースの実態との乖離

### 4. 提案の作成と実施

- 見つかったギャップを優先度順に整理
- 各ギャップに対して具体的な改善案を作成
- ユーザーに提示し、承認後に実施

## 出力フォーマット

```
## ギャップ分析結果

### コードベースの特性
- [言語/フレームワーク/構成の要約]

### 発見されたギャップ

#### ギャップ1: [タイトル]
- **種別**: ルール不足 / スキル不足 / エージェント不足 / 手順・自動化不足 / 設定不整合
- **根拠**: [コードベースのどの特性に対して不足しているか]
- **提案**: [具体的な改善内容]
- **優先度**: 高 / 中 / 低

#### ギャップ2: [タイトル]
...

### カバー済みの領域（参考）
- [現在の設定で十分にカバーされている領域]
```

## 注意事項

- 既存設定を破壊しないよう注意
- ユーザー承認後に変更を実施
- 過剰なルール追加を避ける。本当に必要なものだけを提案する
