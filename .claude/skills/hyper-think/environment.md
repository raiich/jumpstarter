# 動作環境別カスタマイズ

## Claude Code

- **エージェント定義**: `.claude/agents/*.md` を **視点の指示書** として扱う
- **実行方式（R1, R2）**: `Agent` ツールでサブエージェントを **並列に** 起動
- **エージェント指定**: 上記エージェント定義を `subagent_type` で指定
- **コード探索エージェント**: 組み込みサブエージェント `Explore` を `subagent_type` で指定

## GitHub Copilot (CLI / VSCode)

- **エージェント定義**: `.github/agents/*.agent.md`（リポジトリスコープ）or `~/.copilot/agents/*.agent.md`（ユーザースコープ）を **視点の指示書** として扱う
- **実行方式（R1, R2）**: カスタムエージェントは tools としてモデルに提示されるため、メインエージェントが **複数のカスタムエージェントを並列ツール呼び出し** で起動する（Copilot CLI 標準機能）。各サブエージェントは独立コンテキストで実行され、`ideate` の独立性要件を満たす
- **コード探索エージェント**: 組み込みエージェント `Explore` を `agent` ツールで指定（Claude Code 側と同名・同目的）
- **ユーザーが `/fleet` を併用した場合**: タスク分解とプラン化を含む並列ディスパッチに切り替わる。本スキル単独でも並列性は確保されるため `/fleet` は必須ではない
