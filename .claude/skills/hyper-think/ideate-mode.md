# ideate モード

選択肢を発散する。

## 構成

R1 → 要約/同調検知 → R2 → 必要時 R3。

## エージェント選定

R1 は `framework-analyst` を基本に 2〜3 体並列。テーマに応じて以下を追加:

- 機会発見: `constructive-analyst`
- 複数利害関係者: `stakeholder-sim`

`critical-analyst` は R1 では使わず R2 で常時追加投入する（評価を発散に混ぜない）。

`stakeholder-sim` を選んだ場合は [stakeholder-sim.agent.md](../../agents/stakeholder-sim.agent.md) のランダム制約から 2 つを選ぶ（前回と異なる・意外性のある組み合わせを優先）。

## R1 追加指示

判断・評価を保留、量を優先。各エージェントは自身のフレームワーク内で最低 7 個発想。

## 同調検知

R1 完了後、R2 構築前に判定する。次のいずれかで「同調」:

- 主要結論（要約後上位 5〜7 個）の同一カテゴリ・方向性が全体の 70% 以上
- 主要キーワード重複率（上位 10 語）が 3 体間で 60% 以上
- R1 に代替アプローチ・反証がゼロ

## R2 プロンプト

### 発散継続（選定したエージェントが対象）

```
タスク:
1. R1 を 2〜3 視点で再分類してから「乗る」
2. 他エージェントのアイディアに乗って新規を最低 5 個追加（組み合わせ / 拡張 / 反転 / 視点別再分類）
禁止: 既存アイディアの評価・批判、R1 の自分の再掲
新規アイディアのみ報告。
```

### critical-analyst（常時追加投入、adversarial 役）

```
合意点（同調検知の要約 / 未発火時は「なし」と明示）: [...]

タスク: R1 アイディア群と合意点に対して明示的に反対する立場を取る
- 「合意点が間違っているとしたら / R1 が見落としているとしたら」を最低 5 個列挙
- 各エージェントの前提・根拠の脆弱性を指摘
- 反対の立場から見た代替仮説を最低 3 個
禁止: R1 アイディアへの賛同・補強。反対役に徹する。
```

## Round 3: 収束統合（条件付き）

次のいずれかを満たす場合のみメインエージェント自身が実施（サブエージェント起動なし）:

- 同一論点で正反対の結論が 2 体以上、または R2 で未解消トレードオフが 1 件以上
- R1 になかったキーワードを 3 語以上含む新規アイディアが R2 で 1 個以上

内容: 矛盾の解消 / 両立条件の明示 / 合意点の特定 / 優先順位付け。

## 出力フォーマット

アイディアは 20 個以上を目安。

```markdown
## アイディア出し: [テーマ]

### 要約
- 生成数 / クラスタ数 / 中核的な方向性 2〜3 行

### アイディア一覧

#### クラスタ A: [概念名 — 1 行説明]
1. [アイディア] (from: 出典)

### 注目すべきアイディア (3〜5 個)
- #3, #7, #12 — 理由

### 重要論点
- [論点]: 立場 A vs B — 判断条件

### 分析プロセス
- エージェント / ラウンド / 同調検知
```
