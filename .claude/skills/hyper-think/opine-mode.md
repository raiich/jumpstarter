# opine モード

所感・論点・前提を出す。

## 構成

R1 のみ。

## エージェント選定

`critical-analyst` + `constructive-analyst` + `framework-analyst` の 3 体並列。
`framework-analyst` は「初心者の問い」「問題構造の分析」で前提の暗黙化・既知パターン該当を表面化する。

## R1 追加指示

思考過程も含めて報告。

## 出力フォーマット

```markdown
## 所感: [テーマ]

### 結論
2〜3 行。

### 着目した点
- 観点1: [内容] (from: 出典)

### 重要論点・トレードオフ
- [論点]: [どちらにも振り得る理由]

### 検証すべき前提
| # | 前提 | 確認難度 | 決定影響度 | 確認方法 |
|---|---|---|---|---|

### 推奨される次のアクション
1. [行動] — 理由
```
