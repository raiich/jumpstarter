# review モード

既存物（コード・設計・ドキュメント）から問題を抽出する。

## 構成

R1 のみ。同一論点に正反対の結論があれば R2 を実施。

## エージェント選定

1. メインエージェントが [derive-perspectives.md](../sketch-feature/references/derive-perspectives.md) の手順で対象から適用レンズと観点を導出する
2. 導出した観点を以下のエージェントに振り分けて並列起動する

- `critical-analyst`
    - 起動: 常時
    - 渡す: 対象のみ（Devil's Advocate / Pre-mortem）
- `general-purpose`
    - 起動: テスト品質・取りこぼし観点が導出されたとき
    - 渡す: 導出した観点 + 後述の観点ファイル
- `Explore`
    - 起動: 既存実装・テスト基盤との突合が要る観点が導出されたとき
    - 渡す: 導出した観点 + 後述の観点ファイル + 探索方針
- `framework-analyst`
    - 起動: 対象が設計・ドキュメントで、前提の暗黙化または既知パターン該当を疑う観点が導出されたとき
    - 制限: 対象がコードのときは原則使わない
    - 渡す: 対象のみ（初心者の問い / 問題構造の分析）

対象の中身を読まずファイル名・パスのみでレンズを判定しない。判別不能なら構造化ヒアリング。

**観点ファイル**: 観点を網羅したドキュメント
- [testing.md](../sketch-feature/references/testing.md)
- [missable-checklist.md](../sketch-feature/references/missable-checklist.md)

## R1 追加指示

各エージェントに以下を渡す:

- 対象パス（`design.md`・設計スタブ・テストスタブは「パス」のみ渡し、エージェントに読ませる）
- 導出した観点
- 該当する観点ファイル

## R2 プロンプト（矛盾発生時のみ）

```
タスク:
1. 他エージェントが見落とした点を最低 3 個指摘
2. 他エージェントの主張への反論または補強を最低 2 個（根拠付き）
3. 複数分析の組み合わせから生まれる新着想
禁止: R1 の自分の主張のコピペ（新根拠・新事例の追加は可）
```

## 出力フォーマット

```markdown
## レビュー結果

### 対象
- ファイル / 適用レンズ

### 観点
- 適用 / 除外（除外は理由付き）

### 指摘事項
重複統合済み、深刻度 → 影響範囲 → 修正コストで並べ替え。

| # | 深刻度 | 観点 | 指摘内容 | 該当箇所 | 出典 |
|---|---|---|---|---|---|

### クロスレビュー (R2)
矛盾の有無と R2 実施可否を明記（矛盾なしで未実施なら「矛盾なし」と明示し、沈黙と見落としを区別する）。

### 総評
```
