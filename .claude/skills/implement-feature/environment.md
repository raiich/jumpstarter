# 動作環境別カスタマイズ

## Claude Code

- **入力の所在**: 薄い `design.md` は `.local/docs/features/[名前]/`、設計スタブ・テストスタブはビルド対象の実ソースツリー。`sketch-feature` の配置に従う。所在が不明ならユーザーに確認
- **TODO 管理**: `TaskCreate` / `TaskUpdate` / `TaskList`（洗い出した未実装マーカーをユニットとして登録し、着手中・完了を追う）
- **構造化ヒアリング**: `AskUserQuestion`（1 ラウンド最大 4 問）。`design.md` の ❓ 解消・エスカレーション時に使う
- **並列探索**: `Agent` ツール（`subagent_type=Explore`）を `breadth` パラメータで範囲指定して並列に（実装前の現状把握）
- **コンテキスト clear**: `/clear`（無関係タスク間 / 同じ修正で 2 回失敗時 / 準備 → 実装ループの境目）
- **巻き戻し**: `Esc` 2 回 または `/rewind` でチェックポイントから復元（リスクある実装の前にコミットしておく）
- **並列セッション**: git worktree でブランチを分離（`EnterWorktree` / `ExitWorktree`）
- **非対話実行**: `claude -p "<prompt>"`（Fan-out / CI 連携時）

## GitHub Copilot (CLI / VSCode)

- **入力の所在**: 薄い `design.md` は `.local/docs/features/[名前]/`、設計スタブ・テストスタブはビルド対象の実ソースツリー。`sketch-feature` の配置に従う。所在が不明ならユーザーに確認
- **TODO 管理**: メモリ機能で代替（洗い出した未実装マーカーをユニットとして、着手中・完了の状態を明示）
- **構造化ヒアリング**: 通常応答で選択肢を列挙する形に置き換え（1 ラウンド最大 4 問のルールは維持）。`design.md` の ❓ 解消・エスカレーション時に使う
- **並列探索**: 組み込みエージェント `Explore` を `agent` ツールで並列起動。シンプルな範囲では `grep` / `glob` で逐次調査（実装前の現状把握）
- **コンテキスト clear**: 環境のセッションリセット手段（新規 chat 開始等）を用いる
- **巻き戻し**: VCS のコミット境界とエディタの undo を組み合わせる（自動チェックポイントは前提にしない）
- **並列セッション**: git worktree でブランチを分離し、別ターミナル / 別ウィンドウで実行
- **非対話実行**: 環境固有の非対話モード（CLI フラグ等）を利用。利用可否が不明ならユーザーに確認
