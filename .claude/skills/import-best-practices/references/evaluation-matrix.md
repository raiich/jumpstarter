# 評価マトリックス（手段の妥当性評価）

外部情報源（記事、ベストプラクティス、他者の提案等）を取り込む際に、目的とのマッチを評価するマトリックス。

## 判定軸

- **目的との関連性**: 直結 / 関連あり / 関連薄い
- **公式・既存との整合**: 整合 / 部分整合 / 矛盾 / 該当なし
- **現状のカバー状況**: カバー済み / 部分的 / 対応なし

## 判定テーブル例

| 項目 | 目的との関連 | 公式との整合 | 現状 | 判定 |
|---|---|---|---|---|
| 項目 A | 直結 | 整合 | 対応なし | 採用 |
| 項目 B | 関連あり | 部分整合 | 一部 skills で対応 | 採用（差分のみ） |
| 項目 C | 直結 | 整合 | 既存ルールでカバー | カバー済み（取り込み不要） |
| 項目 D | 関連薄い | 公式に該当なし | 対応なし | 対象外 |
| 項目 E | 直結 | 公式と矛盾 | — | 非推奨 |

## 判定の導出

各項目を上の 3 軸で埋め、次の順で最初に該当した判定を一意に採る（空欄を残さない）。判定は出力の検証点であり、全候補項目に付け終えてから取り込み選定へ進む。

1. 公式と矛盾 → **非推奨**
2. 目的との関連が薄い → **対象外**（未対応でも取り込まない）
3. 現状でカバー済み → **カバー済み（取り込み不要）**
4. 上記以外（直結・関連ありで未対応／部分対応）→ **採用**。候補のままでは目的に届かなければ **改変して採用**（→ ミスマッチの扱い）

## ミスマッチの扱い

目的に直結する課題があるのに、取り込み候補がカバーしていない / 手段が最適でない場合:

- 目的を達成するためのより適切な手段を、公式ドキュメントや既存設定の知見から提案する
- 候補項目を改変して取り込む案も含めて検討する

提案形式:

```
目的: 〇〇の改善
元のアプローチ: △△（ミスマッチの理由）
提案: □□（なぜこちらが適切か）
```
