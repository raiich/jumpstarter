# Kaizen

セッションで起きた摩擦（ユーザー指摘・遠回り・ムダ・ストレス）の根本原因を「仕組みの不足」として言語化し、エージェント設定の更新で再発確率を構造的に下げるスキル。

## 実行例

```
❯ /kaizen なぜこうなった?

● kaizen を実行します。セッションの観察となぜなぜ分析を行います。

  ## 観察

  | # | 事象 | 軸 |
  |---|---|---|
  | 1 | ユーザーが「違う」と指摘 | A |
  | 2 | 同じファイルを 3 回読んだ | C |

  ## 問題 1: 同一ファイルの重複読み込み

  1. なぜ? → コンテキストを失った
  2. なぜ? → 探索結果を要約せず再探索した
  3. なぜ? → 「結果が長い時は要約する」ルールが無い
  → 根本原因: 探索結果の要約ルール不在

  ## 仕組み化案

  - 案 A: rules ディレクトリに常時ロードのルールを追加
  - 案 B: スキル化して必要時のみ参照

  ## 適用

  ✅ skills/exploration-summary/SKILL.md を作成
```
