# 出力例

`kaizen` フローの最終出力サンプル。

```
## 観察

| 軸 | 事象 |
|---|---|
| A | ユーザー「テストが set/get の確認ばかりで振る舞いを見ていない」 |
| B | 同じテストファイルを3回読み直し |

## カイゼン対象: 振る舞い検証不足

なぜなぜ:
1. テスト追加時に「何を確認するか」を都度思いつきで決めた
2. テスト設計の起点となる仕様が言語化されていない
3. 機能着手前に振る舞い仕様を明文化する手順が存在しない

根本原因: テスト対象の振る舞い仕様を明文化する設計手順が未定義

対案:
| 案 | 置き場所 | リスク |
|---|---|---|
| 1 | `rules/test-design.instructions.md` 追加 | コンテキスト圧迫 |
| 2 | `skills/<関連スキル>` のテスト設計フェーズに振る舞い列挙を明記 | スキル起動が前提 |
```
