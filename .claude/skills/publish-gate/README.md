# Publish Gate

変更を公開・コミット・プッシュする前に、認証情報・個人情報・ローカルパス・内部 URL などの機微情報の混入を検査するスキル。

## 実行例

```
❯ /publish-gate

● publish-gate を実行します。origin/main 以降の差分を検査します。

  ## 検査結果: 機微情報の混入なし

  origin/main 以降の差分を全カテゴリで走査。ヒットはすべて誤検知または許可対象。

  - 認証情報: ヒットは用語としての credential のみ（許可）
  - 個人識別子・実メール: なし
  - ローカル絶対パス: なし
  - 実 IP: "version": "10.0" がパターンに当たる誤検知のみ
  - アウトバウンド URL: 新規は公式ドキュメント 2 件（許可リスト掲載済み）
  - dev マーカー: なし
```
