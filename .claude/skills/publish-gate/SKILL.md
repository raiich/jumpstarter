# Publish Gate

変更を公開する前に、認証情報・個人識別子・ローカル状態・内部参照・実 IP / URL・dev マーカーなどの機微情報の混入を検査するスキル。設定やテンプレートのように変更が伝播するリポジトリでは、混入が全クローンへ波及するため、公開前に検出して止める。

検査カテゴリ・走査コマンド・許可の判断基準は [references/checklist.md](references/checklist.md) を参照する。動作環境別の挙動は [environment.md](environment.md) を参照（実行前に一度読み込む）。

## フロー

1. **対象範囲を確定** — 既定は公開対象の差分（`origin/main` 以降）。コミット/プッシュ直前なら staged 済み変更、全体監査ならリポジトリ全体を対象にできる。範囲ごとのファイル集合の取り方は [checklist.md](references/checklist.md)
2. **除外を確認** — `.gitignore` で除外されるパス（`.env*` / `*.key` / `*.local.json` 等）は追跡外のため対象外。除外の実効は `git check-ignore` で確認する
3. **パターン走査** — checklist.md のカテゴリごとに対象ファイル集合を走査する。カテゴリは独立に走査でき、網羅・並列の手段は [environment.md](environment.md)
4. **トリアージ** — ヒットは字句一致でなく意味で判定する。許可対象（RFC 文書用レンジ・プレースホルダ・公開ドキュメントの URL）と誤検知（バージョン番号が IP 正規表現に当たる等）を除外し、残りを真の検出として扱う。grep が拾えない残差（バイナリ・難読化・分割された値）は対象に含まれていれば別途目視する
5. **報告** — カテゴリ別に「検出 / 誤検知 / 未対処」を 1 回で報告する。誤検知も「走査したが該当なし」と区別して明示する
6. **対処** — 検出があればプレースホルダへ置換する。既にコミット済みの真の秘密は、追跡から消すだけでは不十分で source 側でローテーションが要る。詳細は [checklist.md](references/checklist.md)

## 注意

- 検出は走査でなくトリアージで決まる。パターンはノイズ前提で、許可対象と照合して切り分ける
- 伝播するリポジトリ（テンプレート・共有設定）では基準を厳しくする。1 クローンで安全でも全クローンに配るに足るかで判断する
- プロジェクト固有の既知安全な参照（自リポジトリの公開 URL・規約上のユーザー名等）は許可リストとして 1 箇所に集約し、走査のたびに再判定しない
- 本スキルは検出と報告が役割。置換は検出を提示したうえで行い、未確認のまま「安全」と断定しない
