# Sketch Feature

機能の追加・改修の設計・テスト仕様を、独立ドキュメントではなく実コードのスタブとコメントで起こすスキル。

## 実行例

```
❯ /sketch-feature リソースを名前で管理する Registry をスタブから作って

● sketch-feature を実行します。薄い design.md + 設計スタブ → テストスタブ を作成します。

  [設計スケッチ] 既存パターン調査 + ヒアリング 1 回（2 問） → 薄い design.md（なぜ・代替案・検証方法）+ registry.go に public な型/シグネチャのスタブ
                （本体は panic("implement me (generated by sketch-feature skill)")、ビルド pass）
  [テストスケッチ] registry_test.go に t.Run ケース 3 件（各 t.Fatal、テストは失敗する）
  → テストスケッチのフェーズ完了（未実装のまま提示。実装は /implement-feature で開始）

  [生成ファイル]
  .local/docs/features/registry/
  └── design.md
  registry/
  ├── registry.go         # 設計スタブ
  └── registry_test.go    # テストスタブ（3 ケース）
```
