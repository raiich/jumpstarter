# 動作環境別カスタマイズ

## Claude Code

- **成果物の保存先**: 薄いドキュメント（`design.md`）は `.local/docs/features/[名前]/`、設計スタブ・テストスタブはビルド対象の実ソースツリー（プロジェクトのレイアウトに従う）。feature 名・配置が不明ならユーザーに確認
- **並列探索**: `Agent` ツール（`subagent_type=Explore`）を `breadth` パラメータで範囲指定して並列に
- **構造化ヒアリング**: `AskUserQuestion`（1 ラウンド最大 4 問）
- **TODO 管理**: `TaskCreate` / `TaskUpdate` / `TaskList`
- **計画モード**: `EnterPlanMode` で読み取り専用の探索に入り、計画確定後に `ExitPlanMode`
- **コンテキスト clear**: `/clear`（無関係タスク間 / 同じ修正で 2 回失敗時 / ヒアリング → 実装の境目）
- **巻き戻し**: `Esc` 2 回 または `/rewind` でチェックポイントから復元（リスクある試行前にコミットしておく）
- **並列セッション**: git worktree でブランチを分離（`EnterWorktree` / `ExitWorktree`）
- **非対話実行**: `claude -p "<prompt>"`（Fan-out / CI 連携時）

## GitHub Copilot (CLI / VSCode)

- **成果物の保存先**: 薄いドキュメント（`design.md`）は `.local/docs/features/[名前]/`、設計スタブ・テストスタブはビルド対象の実ソースツリー（プロジェクトのレイアウトに従う）。feature 名・配置が不明ならユーザーに確認
- **並列探索**: 組み込みエージェント `Explore` を `agent` ツールで並列起動。シンプルな範囲では `grep` / `glob` で逐次調査。広範な探索になりそうな場合は対象範囲をユーザーへ早めに確認
- **構造化ヒアリング**: 通常応答で選択肢を列挙する形に置き換え（1 ラウンド最大 4 問のルールは維持）
- **TODO 管理**: メモリ機能で代替（着手中・完了の状態を明示）
- **計画モード**: 環境固有の plan/preview モードがあれば利用、なければ「先に計画文だけ出力 → 承認 → 実行」の手順で代替
- **コンテキスト clear**: 環境のセッションリセット手段（新規 chat 開始等）を用いる
- **巻き戻し**: VCS のコミット境界とエディタの undo を組み合わせる（自動チェックポイントは前提にしない）
- **並列セッション**: git worktree でブランチを分離し、別ターミナル / 別ウィンドウで実行
- **非対話実行**: 環境固有の非対話モード（CLI フラグ等）を利用。利用可否が不明ならユーザーに確認
