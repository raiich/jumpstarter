# 制御の反転を避ける（処理フローの可読性）

コールバックや「処理を受け取るための interface」を書きたくなったとき、その手前の処理フローに改善余地がないか判断する基準。設計・実装・修正で、読みにくさの原因が制御構造にあるときに参照する。

設計観点全般の導出は [derive-perspectives.md](derive-perspectives.md)、取りこぼしは [missable-checklist.md](missable-checklist.md) を併用する。本ファイルは「誰が誰を呼ぶか」に起因する可読性に絞る。

## 診断: 可読性を下げるのは「制御の反転（push）」

本質は func か interface かではなく、制御の向きにある。呼び出し先が呼び出し元のコードを呼び返す形（push）では、読み手は上から下に追えず、ロジックが渡した処理の中へ飛ぶ。コールバックも「処理を受け取るための interface」も同じ push であり、interface 化しても可読性は変わらない。

interface を一括りにしない。呼び出し元が次を引き出す形（pull）の 1 メソッド interface は直線的で読みやすい。読みにくいのは、呼び出し元のメソッドが呼び返されるハンドラ型（push）。対策は func を interface に変えることではなく、push を pull に戻すこと。

## なぜその処理フローになるか

1. 1 つの関数がオーケストレーション（反復・制御）と要素ごとの処理を両方抱えている → 呼び出し先は生成だけを担い、消費は呼び出し元に持たせる
2. 不透明な手続きの途中に割り込みたい → 手続きを名前付きステップに割り、値を返す形に開いて呼び出し元が合成する
3. setup/teardown で前後を囲みたい → push が正当な数少ないケース（後述）

## 打ち手: pull を優先（上ほど優先）

1. **データを返し、呼び出し元がループ**（既定）。`ForEach(fn)` ではなく結果列を返し、呼び出し元が反復する
2. **フェーズ分割**。中間値に名前を付け、ステップの連なりにする。`Process(in, onResult)` ではなく `a := step1(in); b := step2(a)`
3. **イテレータ**。全件を実体化できない・遅延・巨大・無限のとき。反転をイテレータプロトコルの裏へ隠し、呼び出し元はループとして直線的に読める
4. **ストリーム**。ストリーミングと並行が同時に要るときだけ。ライフサイクル・リーク・キャンセルのコストが付く
5. **データとしての戦略**。「コールバック」が実は設定値の選択なら、列挙型・構造体＋分岐に置き換える

## push（コールバック・ハンドラ）が正当なとき

最も狭い形を選んだ上で、次は push でよい。

- **リソースの囲い込み**: setup/teardown を保証するため呼び出し元のコードを挟む（トランザクション・ロック）
- **実行時の真の多態**: 多数の実装を動的に差し替える
- **本質的に非同期・イベント駆動**: イベント購読・pub/sub
- **ホットパスの push イテレータ**: 中間列の確保が高くつく場合

## Go

- **スライス・map を返す**を既定とする。`for _, x := range f(...)` が最も読みやすい
- **`iter.Seq` / `iter.Seq2`（Go 1.23 以降）** を遅延・巨大・無限列に使う。実体は `func(yield func(V) bool)` という push だが、呼び出し元は `for v := range seq` と pull で読める。反転を言語の `range` の裏へ隠せる
- **チャネル + goroutine** はストリーミング＋並行のときだけ。`context` でのキャンセルと、送信側の終了保証（goroutine リーク対策）をセットにする
- **ハンドラ型と pull 型の interface を区別する**。`io.Reader` / `io.Writer` のように呼び出し元が駆動する 1 メソッド interface は読みやすい。`fn func(T) error` を受け取る形・呼び返されるハンドラ interface は push であり、上の打ち手を先に検討する
- **前後の囲い込みは `defer` を優先**。`WithLock(func(){ ... })` より、リソースを返して呼び出し元が `defer unlock()` する形が直線的なことが多い。解放保証をライブラリ側に寄せたいときだけクロージャ受け取りにする
