# 取りこぼしやすい観点（安全網）

[derive-perspectives.md](derive-perspectives.md) で観点を導出した後に突き合わせる、自力では出にくい取りこぼし項目のトリガー集。
名前中心とし、詳細な解説は付けない（識別に要る短い補足のみ）。常識項目（SQLi・XSS・認可漏れ・N+1 等、導出側で出るもの）は載せない。
該当レンズのみ参照し、各レンズで「該当なし」も明示する。

## セキュリティ

- 入力: 正規化前の比較 / Unicode 正規化 (NFC/NFKC)・双方向制御文字 / 長さプレフィックスを信用した先行確保
- 計算量攻撃: ReDoS / HashDoS / JSON ネスト・XML billion laughs / zip bomb・ZIP slip
- パース: XXE・外部実体 / 危険なパーサオプション既定
- ネットワーク: SSRF → クラウドメタデータ / DNS リバインディング / オープンリダイレクト・Host ヘッダ偽装
- 並行: TOCTOU / メモリ可視性 / ロック順序不一致 / イテレータ無効化 / 取り消し・タイムアウト未設定
- リソース: task 発火後忘れ / 二重 Close・二重 free の冪等性 / 部分初期化失敗のロールバック
- 暗号: IV・nonce 再利用 / 定数時間比較 / AEAD の AAD 取り扱い / 使用後メモリゼロ化 / 証明書検証・ピン留め
- 認可: IDOR/BOLA / 多段認可・状態遷移整合 / CSRF
- 観測: ログへの秘密漏えい（フィールド単位マスク）/ 監査ログの append-only・改ざん検知
- 供給網: typosquatting / postinstall・lifecycle スクリプト副作用 / ロックファイル・署名検証 / SBOM
- FFI・ネイティブ: 整数昇格・符号混在 / sizeof×count オーバーフロー / use-after-free・double-free / strict aliasing / 所有権の GC 跨ぎ
- 言語別:
  - [Go] goroutine リーク・context 伝播
  - [JS/TS] unhandled rejection・prototype 汚染・`__proto__` マスアサイン
  - [.NET] async void・IDisposable・CancellationToken 伝播
  - [C/C++] 例外安全・signal-safety
- secure-by-default: ゼロ値構成が fail-closed か / 危険機能オプトイン・安全機能オプトアウト / スタックトレースの本番露出

## 設計・構成

- モジュール性: 凝集度（単一責務）/ 結合度（最小限の引数）/ 依存方向（一方向・循環なし）/ 置換可能性 / 変更伝播範囲
- パッケージを技術的役割（models/views/controllers）で分割していないか（フィーチャー単位の自己完結を優先）
- 公開境界の最小化（公開しなくてよい型・関数を公開していないか）
- 所有権（誰が確保し誰が解放するか）が API シグネチャから読めるか
- 50 行以下のファイル乱立 / 1 ファイルへの無関連コード混在
- 早すぎる抽象化（3 回登場前の共通化・未使用の拡張ポイント）

## ドメイン別

- **ネットワーク API**: 冪等性キー / 後方互換（フィールド必須化は段階的・未知フィールド無視）/ Deadline の上流伝播 / カーソル vs オフセットの大規模時挙動 / Retry-After・429 / コントラクトテスト
- **ライブラリ・基盤**: スレッド安全性の明示（スレッドセーフ / 単一スレッド / 再入不可）/ deprecation 予告期間 / 性能特性の数値文書化 / 注入リソースを勝手に閉じない
- **サービス・運用**: バックアップは restore 試験まで（取れる ≠ 戻せる）/ RPO・RTO / 単一障害点の洗い出し / 警報疲れ（人を呼ぶ事象に絞る）/ シークレットのローテーション・起動時検証
- **データストア**: lazy loading 由来の暗黙クエリ / EXPLAIN で想定インデックス使用確認 / 大規模 DDL のロック時間見積もり / 移行の冪等性・部分再開 / hot partition・パーティションキー設計
- **UI / フロントエンド**: ローディング閾値（<100ms 不要・>1s 進捗）/ 二重送信防止 / 楽観更新の失敗ロールバック / 大量リスト仮想化 / CLS（画像・フォント）/ prefers-reduced-motion / フォーカストラップと閉じた後の復帰 / localStorage に機密を置かない

## 横断的関心事（該当判定を必ず行う）

- アクセシビリティ (a11y): WCAG（コントラスト・キーボード操作・スクリーンリーダー）
- 国際化 (i18n/l10n): 文字エンコーディング・日付/数値フォーマット・複数形・RTL・タイムゾーン
- コスト: クラウド / 外部 API / ストレージ / 通信量の単価と上限
- データプライバシー: 規制対象データ（GDPR / HIPAA 等）の取得・保存・削除・移転

## ドキュメント

- 前方参照（後続でしか定義しない用語・略語を先行セクションで使用していないか）
- コード例の API が実在し export されているか / バージョン記載がマニフェストと一致するか
- 開発者向け文書で Why（背景・却下した代替案・トレードオフ）が書かれているか
- doc↔code 重複: 仕様・処理手順・mechanism を doc に書いていないか（コメント・擬似コードと重複。doc は why のみ）。「所在は X」「X に従う」式は重複のサイン
- コメント間の重複: package doc が個々の API のコメントを要約・再掲していないか / 別ファイルのコメントに同じ説明がないか。記述は対象に最も近い記述先へ
