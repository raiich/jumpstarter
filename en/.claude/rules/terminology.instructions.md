# Terminology Guidelines

## Usage of "Technology"

In this project, "technology" refers to systematic methods and techniques for solving problems, and is distinguished from specific software, services, or tools.

### Distinction

| Category | Examples |
|----------|---------|
| Technology | Packet routing, name resolution, dynamic programming, inverted indexing, container isolation |
| Software/Tools | Docker, Rails, AWS, Hadoop, Git |
| Specifications/Protocols | TCP/IP, HTTP, SIP |

### Expression Examples

- ❌ "Docker as a technology", "Introduced React technology"
- ✅ "Container technology (implementation: Docker)", "Component-based UI construction approach (adopted by React)"

### Principles

- Do not call tool names "technology"
- Describe the underlying methods and principles as "technology", and reference tools as their implementations
