# Outcome vs Output

Principle of writing goals and metrics as outcomes (results produced) rather than outputs (things built).

## Definitions

- **Output**: what you built or shipped
- **Outcome**: what changed or what value was created as a result

## Conversion Examples

| Output (❌) | Outcome (✅) | How to convert |
|---|---|---|
| Build a CI/CD pipeline | Increase deploy frequency from weekly to daily | The pipeline exists to speed up deploys |
| Create a monitoring dashboard | Cut time-to-detect from X min to Y min | The dashboard exists to catch incidents early |
| Improve documentation | Shorten onboarding from X weeks to Y weeks | Docs exist to accelerate ramp-up |
| Reach 80% test coverage | Reduce post-release defect rate from X% to Y% | Coverage is the means; quality is the goal |

## Conversion Prompt

"If this is achieved, what changes for you or your customers?"

## Applicable Situations

- OKR Key Result descriptions
- "Why" sections of functional requirements
- Task granularity discussions (unit feature vs value)
