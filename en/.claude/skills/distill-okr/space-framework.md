# SPACE Framework — KR Quantification Reference

When struggling to set numeric targets for OKRs, use the 5 dimensions of SPACE to find metric ideas.
No need to cover all dimensions. Use the ones that match your challenges as a starting point.

> Source: "The SPACE of Developer Productivity" (Forsgren et al., 2021)

## 5 Dimensions

### Satisfaction & well-being

Satisfaction with tools, work, and environment.

Metric directions: eNPS, satisfaction surveys, burnout risk (overtime hours, etc.)

### Performance

Outcomes that the system or product delivers to users. Not individual productivity.

Metric directions: reliability (SLO achievement), user experience metrics, customer satisfaction

### Activity

Quantitative aspects of development activity. **Cannot measure productivity alone. Always combine with other dimensions.**

Metric directions: deployment frequency, PR throughput, incident response count

### Communication & collaboration

Quality of information sharing and collaboration.

Metric directions: review response time, onboarding period, knowledge concentration (bus factor)

### Efficiency & flow

The state of uninterrupted focus and smooth value delivery.

Metric directions: lead time, change failure rate, focused work time, handoff count

## How to Use

1. Identify **1–2 dimensions** related to your challenges
2. Use the metric directions as a starting point to think of specific metrics that fit your situation
3. Set current and target values using the KR Quantification Guide (SKILL.md)
